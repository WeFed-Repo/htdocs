SepaPayments

Permits the user to create instructions for SEPA payments.

The entry interface is divided into eight sections:

  • Orderer Data
  • EffectiveDebtorData
  • Beneficiary Data
  • Extra BeneficiaryDebtor
  • EffectiveCreditor Data
  • SEPAPayment Data
  • Beneificiaryresult
  • CVS (1, 2, 3)

Orderer Data

The fields present are as follows:

  • Orderer: select the name (the CUC, Order Descr. e F.C../VAT nr are filled in automatically)
  • CUC:indicate the unique code assigned to the company for identification in the Customer to Business Interaction (CBI) community
  • OdererDescription
  • Fiscal Code/VAT nr
  • Bank
  • Account: select the account (the Orderer, CUC, OrdererDescr., F.C../VAT nr, Bank, IBAN and Unique Code filed will fill in automatically)
  • IBAN

EffectiveDebtor Data

Permits the user to indicate in the case of the effectivedebtornotcoinciding with the debtor the data concerning the Effective Creditor for the SEPA payments. The fieldspresent are:

  • Company Name : indicate the name of the effective debtor
    With the magnifying glass it is possible to look up the name in the records (the Code, Address, Number, Town, Post Code, Prov., Country are filled in automatically). This can also be carried out from Code.
  • Code
  • Type of address: select the type of address. It can be
    • Complete Postal Address
    • Present Address
    • ShipmentAddress
    • ResidencyAddress
    • Postal CorrespondenceAddress
    • Post Box
  • Address
  • Number
  • Town
  • Post Code
  • Prov.
  • Country

SEPA Payments Data

The fields present are:

  • Amount: amount of the instruction
  • Currency
  • PaymentDescription
  • Cause: select the CBI cause of the payment, choosing from
    • GenericPayment - 48000
    • ComplimentaryInsurancePayment- 48015
    • Merchant Payment- ZZ000
  • Purpose of coded payment: select the purpose of the payment choosing from those available
  • ISO Cause: select the ISO cause choosing between those available
  • Purpose of payment: (alternative Purpose of coded payment ) indicate the purpose of the payment
  • Matching information: indicate any extra information or details

Result for Benefiary

The fields present are:

  • Result for Beneficiary: if selected it is necessary to fill in the Subject Name, Result Method and CUC/Email fields
  • SubjectName
  • Resultmethod: select the way to comunicate the result to the Beneficiary. It can be
    • CUC
    • E-Mail


  • CUC/E-Mail


CVS (1, 2, 3)

The CVS sections contain fields for entering data for the Communication of Evaluation Statistics concerning the collection. The fieldspresent are:

  • Evaluation Cause: select the evaluation cause from those available
  • CVS Type: select the type of evaluation statistic communication. It can be
    • Below CVS limit
    • Subjectnotresident
    • Assignevaluation cause
  • CVSAmount
  • Currency

It is possible to enter up to three CVS (each one is accessible by clicking on the arrows under the relative heading): for each one it is necessary to fill in the specific fields described above.

Click on Cancel to annul the entry operation.

Click on the Enter button to confirm the entry on the instruction. The procedure offersthreeoptions:

  • New instruction: the page for the entry of a new instruction;
  • New instruction with previous data: the entry page with data previously entered is viewed.

    N.B. This function does not modify the data entered in the previous instruction, but allows the user to copy into the new page the same data without having re-enter it.
  • Create order: the Create Order page opens automatically.
