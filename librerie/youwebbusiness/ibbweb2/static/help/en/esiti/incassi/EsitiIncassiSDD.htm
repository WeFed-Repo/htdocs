Collection Results
In this page itispossible to view the results of Direct DebitCollections.

The information is obtained by setting filter parameters (Filter Collection Results), it is possible to choose:

  • Company name/Account holder: In the case of the user profile only being able to operate for one company then this field appears filled in with the relative Company name (the SIA is filled in automatically)
  • Sia: In the case of the userprofileonlybeingable to operate for one company thenthisfieldappearsfilled in with the associated Sia code
  • Bank: In the case of the userprofileonlybeingable to operate for onebankthenthisfieldappearsfilled in with the relative ABI code and description
  • Account: In the case of the userprofileonlybeingable to operate for oneaccountthenthisfieldappearsfilled in with the relative CAB code and account number
  • Type of Collection: allows the user to choose the type of collection. It can be
    • Ordinary
    • Express
    • Ordinary COR1
  • Collection Status: allows the user to choose the instruction status to view. It can be
    • Signed
    • Confirmed
    • Rejected
  • Collection Sequence: allows the user to filter information according to the collectionsequencespecified in the originalinstruction. It can be
    • FRST -First of a series of collections
    • RCUR -For a series of collections
    • FNAL -Last of a series of collections
    • OOFF -Single notrepeated
  • Send date From: and To
  • Expiry date From: and To
  • Amount From: and To
  • Debtor: andTo

Clicking on the Search Order box or Search Sub-order box activates the following fields, useful for filtering information in data contained in collection orders sent or relative sub-orders.

  • Send date From: and To: (Advanced section): permits the filtering of information for different time intervals concerning the send date of the order or sub-order.
  • Execution date From: and To: (Advanced section): permits the filtering of information for different time intervals concerning the execution date of the order or sub-order.
  • Amount From: and To: (Advanced section): permits the filtering of information for different ranges of amountin the order or sub-order

It isalso possible to set the number of rows viewed per page, by choosing one of the numbers in the corresponding field.

Click on View to obtain the results.

The information isstructuredasfollows:

  • Data:it can be:
    • Instruction: Instructionsent from the Payment Area for which a resulthasbeenrequested
    • Result matched to instruction:instructionsent from Payment Area hasbeenmatched to a result
    • Result: the instruction result is not associated


  • Type of collection.: indicates the type of collection. It can be
    • Ordinary
    • Express
    • Ordinary COR1
  • Orderer
  • Debtor
  • Expiry: where present, indicates the date of expiry specified in the instruction
  • File date
  • Amount
  • Currency
  • Collection status: indicates the status of the collection. It can be
    • Signed
    • Confirmed
    • Rejected

Where available, in the column it is possible to apply an ascending or descending order by clicking on the arrows present below the column heading; the results by default are ordered according to the File Date column.

The Detail of Instruction/Result is accessed from the Instruction Status column

Navigation Tools and Common Functions

It is possible to use navigation tools and the common functions available: to choose the number of results viewed per page, to filter the results using the drop down menus, to return to the filter home page, to reset the filters previously set, view the current data in a PDF format (which can be printed or saved to be used off-line).
