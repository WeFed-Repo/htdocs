Direct DebitCollections

Allows the creation of SDD collection instructions.

The entry interface is divided into eleven sections

  • Orderer Data
  • Effective Creditor Data
  • Debtor Data
  • Extra Information Debtor
  • EffectiveDebtor Data
  • Extra Information regarding mandate
  • Orginal Creditor Data
  • OriginalDebtor Data
  • Collection Data
  • CVS (1, 2, 3)
The Original Creditor Data and Original Debtor Data are viewed if the Extra Information Mandate Section is expanded.

Orderer Data

The fields present are as follows:

  • Orderer: select the name (the CUC, Order Descr. e F.C../VAT nr are filled in automatically)
  • CUC:indicate the unique code assigned to the company for identification in the Customer to Business Interaction (CBI) community.
  • Orderer Description
  • F.C../VAT nr
  • Bank
  • Account: select the account (the Orderer, CUC, OrdererDescr., F.C../VAT nr, Bank, IBAN andUnique Code filed will fill in automatically)
  • IBAN
  • IBAN Commissions if more than one account is active for the ABI selected choose the account to use to debit any commissions/charges concerning the transaction requested. With the IBAN commissions button it is possile to view further information
  • Creditor Identification: indicate the code number supplied by ABI, unique for the creditor

Collection Data

The fieldspresent are:

  • Amount: amount of the instruction
  • Currency
  • Collection Description: with the Collection Description button it is possible to find out useful information about filling in this field.
  • Expiry Date
  • Type of Collection: select the type of collection between
    • Ordinary
    • Express
    • Ordinary CORE
  • Collection Sequence: select the type of collection sequence. Can be:
    • FRST - First of a series of collections
    • RCUR - For a series of collections
    • FNAL - Last of a series of collections
    • OOFF -Singularnotrepeated
  • ISO Cause: select the ISO cause choosing between those available
  • Purpose of coded payment: select the purpose of the payment choosing between those available
  • Matching information: indicate any extra information or details

CVS (1, 2, 3)

The CVS sections contain fields for entering data for the Communication of Evaluation Statistics concerning the collection. The fieldspresent are:

  • EvaulationCause: select the evaluation cause from those available
  • CVS Type: select the type of evaluation statistic communication. It can be
    • Below CVS limit
    • Subjectnotresident
    • Assignevaluation cause
  • CVSAmount
  • Currency

It is possible to enter up to three CVS (each one is accessible by clicking on the arrows under the relative heading): for each one it is necessary to fill in the specific fields described above.

Click on Cancel to annul the entry operation.

Click on the Enter button to confirm the entry on the instruction. The procedure offersthreeoptions:

  • New instruction: the page for the entry of a new instruction;
  • New instruction with previous data: the entry page with data previously entered is viewed.
    N.B. This function does not modify the data entered in the previous instruction, but allows the user to copy into the new page the same data without having re-enter it.
  • Create order: the Create Order page opens automatically.
